# ClassicMap의 연주 비교는 왜 DTW가 필요했나

클래식 음악에서 같은 곡은 같은 시간이 아니다.

악보는 같아도 연주자마다 템포, 루바토, 프레이징이 다르다. 어떤 연주는 한 구간을 빠르게 지나가고, 어떤 연주는 같은 구간을 오래 끌고 간다.

그래서 “1분 20초부터 2분 10초까지 비교” 같은 방식은 금방 깨진다.

## 문제

사용자가 원하는 건 단순 동시 재생이 아니다.

같은 악장, 같은 주제, 같은 프레이즈를 서로 다른 연주에서 맞춰 듣는 것이다.

이 요구사항은 UI 문제가 아니라 매칭 문제다.

- 입력은 서로 다른 YouTube 영상이다.
- 같은 곡이어도 길이가 다르다.
- 시작 지점과 섹션 경계가 다르다.
- 사람이 수동으로 맞추면 데이터가 커질수록 유지가 안 된다.

## 선택

DTW, Dynamic Time Warping을 사용하면 서로 다른 속도로 진행되는 시퀀스를 정렬할 수 있다.

ClassicMap에서는 음악 특징을 뽑고, 템포 차이를 견디는 방식으로 대응 구간을 찾는다. 핵심은 “같은 시간”이 아니라 “같은 음악적 위치”를 찾는 것이다.

## 배운 점

좋은 기능은 종종 화면 밖에서 결정된다.

겉으로는 비교 플레이어 하나지만, 실제 품질은 데이터 수집과 구간 매칭에서 나온다. 그래서 포트폴리오에서 이 프로젝트를 설명할 때도 화면보다 알고리즘 선택 이유를 먼저 보여줘야 한다.
