# 블로그 작성 가이드

> 이 노트는 blog 루트에 있어서 sync 대상이 아님 (갈래 폴더 안의 노트만 스캔됨). `publish: false`는 이중 안전장치.

## 발행 흐름

1. 갈래 폴더(`설계와 구현` / `생각과 질문` / `읽기와 보기` / `견문과 모임` / `해의 회고`) 안에 노트 작성
2. Properties에 `publish` 체크박스 추가하고 켜기
3. 터미널에서 `blogpub` → 끝 (몇 분 뒤 kang1027.com/blog + Bear 반영)

---

## 새 글 프로퍼티

이 노트의 Properties가 곧 템플릿 — 새 글 만들 때 그대로 따라 적으면 됨 (전부 옵션, `publish`만 필수).

- 쓰는 동안은 `publish` 꺼둠(초고), 내보낼 때 켜고 `blogpub`
- `date`는 발행일로 — 안 적으면 파일 생성일이 들어감
- `slug`는 URL용 영문 권장 (`/blog/english-url-slug`). 안 적으면 파일명 그대로

## 본문에서 먹는 문법

### 텍스트

- **굵게** `**굵게**` / *기울임* `*기울임*` / ~~취소선~~ `~~취소선~~`
- 제목은 `##`부터 (`#` 한 개짜리는 글 제목과 겹쳐서 자동 제거됨)

### 코드

인라인 코드는 `` `백틱` ``, 블록은 언어 표기와 함께:

```ts
const greet = (name: string) => `안녕, ${name}`;
```

### 하이라이트

==형광펜 하이라이트== — 옵시디언에서 보이는 그대로 블로그에서도 노란 마커로 나옴.

### 콜아웃

옵시디언 콜아웃 그대로 쓰면 잉크 계열별로 먹 디자인이 다른 수묵 박스로 렌더됨. 전 타입:

> [!note] 메모 — `note`
> 먹/백 잉크(테마 반전) + 번짐 텍스처. 안에서 **굵게**, `코드`, 링크 전부 됨.

> [!quote] 인용 — `quote`
> 먹/백 잉크 + 산수 안개 텍스처.

> [!example] 예시 — `example`
> 먹/백 잉크 + 번짐.

> [!abstract] 요약 — `abstract` (`summary`)
> 먹/백 잉크 + 번짐.

> [!tip] 팁 — `tip` (`hint`)
> 청 잉크 + 유체 흐름 텍스처.

> [!info] 정보 — `info`
> 청 잉크 + 유체 흐름.

> [!todo] 할 일 — `todo`
> 청 잉크 + 유체 흐름.

> [!warning] 주의 — `warning` (`caution`)
> 황토 잉크 + 거친 붓터치 텍스처.

> [!danger] 위험 — `danger` (`error`)
> 인주 잉크 + 튄 자국 텍스처.

> [!question] 질문 — `question`
> 인주 잉크 + 튄 자국.

> [!success] 완료 — `success`
> 송록 잉크 + 잔잔한 결 텍스처.

제목 생략하면 타입명이 들어가고, 모르는 타입은 note로 폴백.

### HTML 태그

<u>밑줄</u>(`<u>`), `<br>`, `<details>` 같은 인라인 HTML도 렌더됨.

### 링크 · 이미지 (옵시디언 방식 그대로)

- 외부 링크: `[텍스트](https://...)` — 자동으로 새 탭
- 이미지: `![[첨부.png]]` — sync가 알아서 복사하고 경로 변환 (vault 어디 있든 찾음)
- 다른 글 링크: `[[발행된 노트명]]` — 그 글도 발행 상태면 블로그 링크로 변환, 아니면 일반 텍스트
- 주석: `%%이 안의 내용%%` — 발행 시 통째로 제거 (메모용)

## 안 먹는 것 (쓰면 그대로 텍스트로 노출되거나 무시됨)

- **열 나눔** — 아직 미구현
- 발행 안 된 노트로의 `[[링크]]` — 링크가 아니라 일반 텍스트로 변환됨
