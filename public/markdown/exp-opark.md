# (주)오파크 — 풀스택 개발자

**기간**: 2026년 ~ 현재 (재직중)
**소속**: (주)오파크
**역할**: 풀스택 개발자

## 회사 / 제품 소개

(주)오파크는 건축·인테리어 산업의 자재 거래를 디지털로 연결하는 스타트업입니다. 두 개의 주력 서비스를 운영하고 있습니다.

- **[TOONT-M](https://toont-m.com)** — AI 기반 건축 자재 매칭 플랫폼 (B2B SaaS)
  - 이미지/텍스트 기반 자재 검색
  - 건축·인테리어 전문가 ↔ 자재 공급사 매칭
  - 발주·배송·시공까지 아우르는 프로젝트 관리
- **TOONT** — 자체 운영 쇼핑몰 (toont.com)

## 담당 업무

### 1. TOONT-M 플랫폼 전반 개발

회사의 핵심 SaaS인 TOONT-M의 전반적인 개발에 참여하고 있습니다.

- 프론트엔드 / 백엔드 / 데이터 모델링까지 도메인 단위로 책임지고 구현
- 자재 검색·매칭·프로젝트 관리 등 핵심 기능의 신규 개발과 개선
- 운영 중 발생하는 이슈 대응 및 데이터 정합성 관리

### 2. SMS ↔ Slack 게이트웨이 구축

CS 운영팀이 사용하던 **CS폰의 SMS 송·수신을 Slack에서 처리할 수 있도록** 자체 게이트웨이 서비스를 설계하고 구축했습니다.

- CS폰으로 들어오는 SMS를 실시간으로 Slack 채널에 전달
- Slack에서 답장 메시지를 작성하면 CS폰 번호로 SMS 발신
- CS 응대 이력의 추적 가능성 / 다인 협업 / 부재 대응을 한 번에 해결
- 기존 휴대폰 한 대에 묶여 있던 CS 워크플로우를 팀 단위로 확장

### 3. TOONT 쇼핑몰 리뉴얼

자체 쇼핑몰 [toont.com](https://toont.com)의 리뉴얼 프로젝트 전반을 담당했습니다.

- 사이트 구조·디자인·기능 리뉴얼 전반 책임
- **결제 시스템 관리** — PG 연동, 결제 플로우 점검, 운영 이슈 대응
- **Meta Pixel 관리** — 이벤트 설계, 전환 추적, 광고 성과 측정 인프라 구성
- 마케팅 / 운영 부서 요구사항 반영 및 데이터 연동

## 사용 기술 / 일하는 방식

- **Backend**: Rust 기반 비동기 서버 / 외부 API 연동 (메시징, PG, 이미지/AI 검색)
- **Frontend**: React, React Native, Flutter
- **운영**: 셀프 호스팅 환경에서의 배포 / 모니터링 / 장애 대응
- **워크플로우**: Claude Code를 적극 활용한 AI 페어 프로그래밍

## 지향하는 방향

- **만들고 끝**이 아니라 **결제·메시지·광고 픽셀까지 직접 책임**지는 풀스택
- 운영팀 워크플로우를 코드로 풀어내 **사람이 매여 있던 일을 시스템으로 옮기는 것**에 흥미가 큼
- 작게 자주 배포하고, 사용 데이터에 반응해 빠르게 다듬는 방식을 선호
