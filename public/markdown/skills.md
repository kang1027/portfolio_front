## Core Skills

### 🦀 Rust - (4/5)

근래 가장 집중하고 있는 언어입니다. OmniNews와 ClassicMap 프로젝트의 백엔드를 Rust로 개발했고, Rust 오픈소스 생태계에 기여하고 있습니다.

Rocket 프레임워크로 비동기 API 서버를 구축하고, sqlx로 MariaDB와 연결해서 실제 서비스를 운영하고 있습니다. Apple Music API, Spotify API, KOPIS API 등 외부 API를 호출하고 데이터를 가공하는 작업도 다수 진행했고, WebSocket으로 포트폴리오에 실시간 음악 재생 정보를 연동하기도 했습니다.

특히 옴니뉴스를 개발하며 RRSS(Reverse-RSS)를 만들 때 rannoy(Annoy)와 rust-bert를 사용해 데이터를 임베딩하고 유사도 검색 알고리즘을 구현해 머신러닝 분야도 다루었습니다.

처음 Rust를 학습할 때는 [O'REILLY Programming Rust]교재와 [Privaxy](https://github.com/barre/privaxy) 프로젝트를 클론 코딩하며 Rust언어의 구조와 인증서 관리, HTTP/Proxy 통신을 학습했습니다.

### ⚛️ Flutter & React & React Native - (3/5)

프론트는 하나의 코드로 iOS, Android, Web을 모두 커버하는 크로스 플랫폼 개발을 선호합니다.

**OmniNews**는 Flutter로 iOS/Android를 개발하고, 추후 React+Vite로 웹을 따로 개발했습니다. In-App Purchase로 구독 시스템을 구현하고, AdMob, Firebase notification등을 사용해 앱 출시의 전 과정을 직접 진행했던 프로젝트입니다.

**ClassicMap**은 Expo를 사용해 개발했고, iOS/Android/Web을 하나의 코드베이스로 동시에 개발할 수 있었습니다. 클래식 음악 정보를 제공하는 앱이다 보니, Artists와 Composers 정보를 위키피디아에서 자동으로 크롤링해서 DB에 넣는 매크로도 개발했습니다.

또한, performances comparison을 DTW(Dynamic Time Warping) 알고리즘을 사용해 YouTube 영상의 구간을 자동으로 매칭하는 프로그램을 개발했습니다.

- 예를 들어 임윤찬의 라흐마니노프 협주곡 3번 연주 영상에서 3악장 클라이맥스 부분(37:48~42:32)을 입력하면, 다른 연주자들(랑랑, 유자 왕 등)의 같은 구간을 자동으로 찾아줍니다.
  - 랑랑(33:59 ~ 38:48), 유자 왕(38:37 ~ 43:38)

### 🐳 Docker & 인프라 관리 - (3/5)

제가 개발한 프로젝트들은 Docker Container로 관리하며 온프레미스로 운영하고 있습니다. Caddy로 리버스 프록시를 관리하고 있고, Cloudflare로 CDN과 보안 설정을 관리하며, 서버 네트워크와 방화벽도 직접 설정하고 있습니다.

원래 AWS나 Google Cloud를 사용했으나, 개인 프로젝트는 온프레미스로 직접 관리하는 게 비용절약 및 제어권의 이점이 있어 마이그레이션해 운영중입니다.

### 🗄️ 데이터베이스 설계 & 운영 - (4/5)

MySQL과 MariaDB로 RDB를 설계 및 운영했고, WISEFEE당시 Redis로 캐싱을 구현했으며, TokeNest를 개발할 땐 NoSQL인 MongoDB를 사용했습니다.

특히 주성엔지니어링에서 근무하며 대규모 ERP/PLM 데이터베이스를 관리할 수 있었습니다. 수천개 이상의 테이블의 연관관계를 파악하고 쿼리를 작성해보며 효율적인 쿼리문을 작성하는 실무 경험을 쌓았습니다.

### ⛓️ 블록체인 & 스마트 컨트랙트 - (2/5)

오픈SW공모전을 준비하며 Klaytn 프로젝트를 포크해 Solidity로 스마트 컨트랙트를 작성해서 TokeNest 전용 풀을 구성했습니다. 원자재 가격 변동성 풀을 구현하며 블록체인 기반 시스템을 이해할 수 있었습니다.

요즘은 Aave, GMX, HyperLiquid 같은 DeFi 플랫폼에서 직접 자산을 관리하며, 빠르게 변화하는 블록체인 시장을 이해하고 있습니다.

# Development Style

### 🔐 보안 중심 개발

정보보안기사 자격을 취득해 기본적인 서비스 취약점 및 보호대책을 인지하고 있으며, Rust의 언어적 성능과 포인터 관리의 이점을 활용해 개발하고 있습니다.

서버 보안 설정과 HTTPS 인증서 관리, 방화벽 설정 등 실제 운영 환경에서 프로젝트를 설정하고 관리하고 있습니다.

근래엔 kalilinux를 사용해 해킹과 보안과 관련해 직접 실습하며 공부하고 있습니다. 단순 네트워크 상의 보안뿐 아닌, 일상생활에서의 보안 대책도 강구하고 있습니다.

### 🚀 린 개발론 - 빠르게 만들고, 빠르게 실패하고, 빠르게 배우기

개인 프로젝트를 진행하며 린 개발 방법론을 따르고 있습니다. 새로운 프로덕트를 구상할 때 완벽한 기획을 세우고 시작하기보다는, 빠른 MVP를 만들어 실제로 운영해보고 피드백에 따라 수정하는 방법으로 개발하고 있습니다.

OmniNews를 개발하며 린 개발론의 필요성을 느꼈고, ClassicMap을 개발하며 린 개발론을 적극적으로 적용했습니다. 필요한 핵심 기능만을 빠르게 개발하고 피드백을 통한 고도화 개발로 아이디어 구상부터 앱스토어 출시까지 **한 달**에 걸쳐 프로젝트를 완성했습니다.

"완벽하고 예쁜 코드"보다 "동작하는 서비스"와 "가치있는 프로덕트"가 더 중요하다고 생각합니다. 실패하더라도 빠르게 실패하고, 이를 통해 배운 내용을 다음 프로젝트에 바로 적용하는 식으로 개발하며 성장하고 있습니다.

린 개발론에 따라 AI도 적극적으로 활용하고 있습니다. Rust, React Native, Flutter, Docker, 블록체인 같은 다양한 기술 스택을 AI를 접목해 빠르게 학습할 수 있었고, 강의를 수동적으로 따라하는 게 아닌, 서비스를 실제로 만들고 문제에 부딪히고 해결하는 저만의 학습 방법을 구축할 수 있었습니다.

---

**실제 서비스를 만들고 운영하는 경험**을 통해 단순히 코드를 작성하는 것을 넘어, 기획부터 배포, 운영까지 전체 라이프사이클을 이해하고 있습니다. 새로운 기술을 탐구하고 실전에 적용하는 것을 즐기며, 항상 더 나은 개발 방법을 고민합니다.
