## Design Context

### Users
Portfolio visitors, recruiters, collaborators, and technically curious readers who want to understand how Kang builds and reasons through products, not just scan a resume.

### Brand Personality
Native, personal, practical. The interface should feel like a lived-in macOS workspace: direct, polished, slightly opinionated, and focused on real work artifacts.

### Aesthetic Direction
The Bear app is the portfolio-native writing surface: Posting should browse like a note library with project threads and nested notes. The public `/blog` path should stand as a real writing site, closer to a quiet archive than an app tab: typography first, project/year hierarchy, minimal icons, and a dark-first theme derived from Bear's restrained red and graphite neutrals.

### Design Principles
- Keep the desktop portfolio illusion intact inside Bear; use grouped note navigation rather than one flat blog list.
- Make the external blog feel like a proper site, not a copied tab: no empty app sidebars, no card-heavy feed, no decorative icon grid.
- Treat posts as threads with metadata: project group, category, tags, date, summary, and related work should be scannable without visual noise.
- Use the existing Bear/macOS visual language first: sidebars, lists, focused reading panes, restrained red accent, native spacing.
- Write and display content as engineering judgment and work notes, not generic tutorials.
- Prefer text structure, dates, section rules, and archive hierarchy over icons.
- Prefer accessible links, buttons, focus states, and readable article measure over decorative density.
