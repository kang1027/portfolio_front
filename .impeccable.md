## Design Context

### Users
Portfolio visitors, recruiters, collaborators, and technically curious readers who want to understand how Kang builds and reasons through products, not just scan a resume.

### Brand Personality
Native, personal, practical. The interface should feel like a lived-in macOS workspace: direct, polished, slightly opinionated, and focused on real work artifacts.

### Aesthetic Direction
The Bear app is the primary writing surface. Blog browsing should feel like opening a notes library inside the existing macOS portfolio, with tags, note lists, and readable Markdown articles. The public `/blog` path is a shareable permalink surface, not a generic marketing blog.

### Design Principles
- Keep the desktop portfolio illusion intact; blog UI should look like part of the operating system, not a separate website bolted on.
- Treat posts as notes with metadata: category, tags, date, summary, and related work should be scannable without visual noise.
- Use the existing Bear/macOS visual language first: sidebars, lists, focused reading panes, restrained red accent, native spacing.
- Write and display content as engineering judgment and work notes, not generic tutorials.
- Prefer accessible links, buttons, focus states, and readable article measure over decorative density.
